## 1️⃣ 골든 패스 — 핵심 사용자 여정

> 신규 유저가 앱을 처음 실행하여 로그아웃 후 재로그인하기까지의 전체 플로우 검증

| # | 테스트명 | 검증 내용 | 결과 |
|---|---------|----------|:----:|
| 1 | 스플래시 → 온보딩 | 앱 진입 시 스플래시(2.5s) 후 온보딩 화면 전환 | ✅ |
| 2 | 온보딩 슬라이드 | 4단계 슬라이드 완료 → 로그인 화면 도달 | ✅ |
| 3 | 회원가입 (Step 1) | 이메일 중복확인·비밀번호·건강 기본정보 입력 | ✅ |
| 4 | 캐릭터 생성 | 알 탭 → 부화 애니메이션 → 이름 입력 → 홈 도달 | ✅ |
| 5 | 홈 화면 | Streak·오늘의 미션·오늘의 추천 표시 확인 | ✅ |
| 6 | 미션 화면 이동 | "전체보기" 클릭 → 미션 화면 진입 | ✅ |
| 7 | 식단 기록 화면 | 하단 탭 "식단" 클릭 → 식단 화면 진입 | ✅ |
| 8 | 재로그인 | 로그아웃 후 이메일/비밀번호 재로그인 → 홈 복귀 | ✅ |

**주요 기술 이슈 해결 내역**
- `test.describe.serial` 적용 → 순차 실행으로 백엔드 Rate Limit(HTTP 429) 방지
- 알 이미지 bounce 애니메이션 → `force: true` 클릭으로 해결
- 분석 애니메이션 6초 대기 → `toBeHidden` 전환 감지로 타이밍 안정화

## 2️⃣ 유저 타입별 미션 검증

> 건강정보 입력값에 따라 6가지 유저 타입이 올바르게 분류되고, 각 타입에 맞는 미션이 제공되는지 검증

### 유저 타입 분류 로직

```
당뇨 선택 → 1형 당뇨  ────────────────────────────────── diabetic_1
         → 2형 당뇨  ────────────────────────────────── diabetic_2
         → 없음 ─── BMI ≥ 25 또는 나이 ≥ 45 ────────── at_risk
                  └── 정상 → 분석 화면 목표 선택
                      ├── 체중 감량 및 다이어트 ────── general_diet
                      ├── 체력 증진 및 건강 개선 ──── general_health
                      └── 근력 증진 및 몸만들기 ────── general_fitness
```

### 테스트 결과

| # | 유저 타입 | 테스트 조건 | 고유 미션 검증 | 공통 미션 검증 | 결과 |
|---|----------|-----------|:------------:|:------------:|:----:|
| U-01 | `general_diet` | 나이 25세, BMI 20.8 → 다이어트 목표 선택 | 유산소 운동 30분 | 만보 걷기·물 마시기 | ✅ |
| U-02 | `general_health` | 나이 30세, BMI 20.2 → 건강 개선 목표 선택 | 채소 섭취 | 만보 걷기·물 마시기 | ✅ |
| U-03 | `general_fitness` | 나이 28세, BMI 22.9 → 몸만들기 목표 선택 | 근력 운동 45분 | 만보 걷기·물 마시기 | ✅ |
| U-04 | `at_risk` | 나이 35세, BMI 27.7 (≥25) → 위험군 자동 판정 | 식후 걷기 | 만보 걷기·물 마시기 | ✅ |
| U-05 | `diabetic_1` | 1형 당뇨 선택 → 분석 화면 없이 바로 권한 | 인슐린 주사 체크 | 만보 걷기·물 마시기 | ✅ |
| U-06 | `diabetic_2` | 2형 당뇨 선택 → 분석 화면 없이 바로 권한 | 하체 운동 | 만보 걷기·물 마시기 | ✅ |

**검증 포인트**
- 당뇨 유저(U-05/06): 분석 화면 스킵 → 바로 권한 화면으로 이동 확인
- 위험군(U-04): BMI 기반 자동 판정 → "위험군 건강 관리 시작하기" 버튼 표시 확인
- 공통 미션(만보 걷기·물 마시기)은 모든 타입에서 제공됨

| D-02 | 끼니 탭 전환 | 아침→점심→저녁→간식 클릭 시 해당 탭 활성화 및 업로드 헤더 변경 | ✅ |

## 🔍 테스트 수행 중 발견된 기술 이슈

| 유형 | 내용 | 해결 방법 |
|------|------|----------|
| **API Rate Limit** | 병렬 실행 시 백엔드 HTTP 429 오류 | 모든 spec에 `test.describe.serial` 적용 |
| **localStorage 의존성** | 로그인만으로는 미션 미생성 (`setUserProfile` 필요) | 미션 테스트는 신규 회원가입 플로우로 진행 |
| **애니메이션 불안정성** | 알 이미지 bounce 애니메이션으로 클릭 불가 | `click({ force: true })` 적용 |
| **Strict Mode 위반** | 동일 텍스트 다수 존재 시 Playwright 오류 | `{ exact: true }` + `.first()` 적용 |
| **분석 화면 대기** | AI 분석 애니메이션 6초 → 고정 대기 부족 | `toBeHidden()` 전환 감지 방식으로 변경 |
| **캐릭터 탄생 타이밍** | 버튼명 오인식 + 화면 전환 타이밍 불일치 | 정확한 placeholder/버튼명 사용 + 2.5s 대기 |
| **식단 결과 텍스트** | `분석 결과` 대신 음식명이 헤더에 표시됨 | `저장하기` 버튼 가시성으로 완료 상태 감지 |
